# Ejercicio Nro: 1
## Enunciado

¿Qué sabés de metodologías ágiles?

1. ¿Qué conceptos sobre metodologías ágiles aprendiste en otras asignaturas?
2. ¿Estás trabajando actualmente? En caso afirmativo, ¿qué metodología utilizan en la empresa?
3. ¿Qué herramientas de integración y entrega continua conocés o utilizás?
4. ¿Cuáles considerás que son las características principales de un desarrollo ágil?

## Resolución

1) A partir de distintas materias, se puede entender que una metodología de desarrollo es un marco de trabajo que permite organizar, planificar y controlar el desarrollo de un sistema mediante herramientas, procesos y roles .

También se relaciona con el concepto de ciclo de vida del software, que incluye etapas como:

- Relevamiento
- Análisis
- Desarrollo
- Pruebas
- Despliegue

En las metodologías ágiles, este ciclo no es lineal, sino iterativo, es decir, se repite en ciclos cortos donde se mejora continuamente el producto.


2) Actualmente no estot trabajando en ningun entorno de desarrollo de software, pero en la industria es común el uso de metodologías ágiles como Scrum.

Scrum es un framework ágil que organiza el trabajo en ciclos cortos llamados sprints, permitiendo entregas parciales y mejora continua .

Estas metodologías se usan porque permiten:

- Adaptarse a cambios
- Entregar valor rápido
- Mejorar la colaboración en equipo


3) En un contexto ágil, es común utilizar herramientas que automatizan procesos de desarrollo, conocidas como CI/CD (Continuous Integration / Continuous Delivery).

Algunos ejemplos:

- GitHub / GitLab (control de versiones)
- Jenkins / GitHub Actions (automatización)
- Docker (contenedores)
- Azure DevOps

Estas herramientas permiten:

- Integrar cambios de código constantemente
- Detectar errores temprano
- Automatizar pruebas y despliegues

Esto se alinea con Agile porque favorece la entrega continua de valor al cliente, uno de sus principios clave.


4) Las metodologías ágiles se basan en un enfoque flexible y adaptativo. Sus principales características son:

- Iteración e incrementalidad:

El producto se desarrolla en pequeñas partes (sprints), no todo de una vez.


- Adaptación al cambio:

Los requisitos pueden cambiar durante el proyecto y se ajustan continuamente.


- Entrega continua:

Se entregan versiones funcionales de manera frecuente.


- Trabajo en equipo:

Fuerte colaboración entre desarrolladores y clientes.


- Enfoque en el cliente

La prioridad es satisfacer al cliente mediante entregas tempranas y continuas.


- Mejora continua

Se revisa constantemente el trabajo para optimizar procesos (inspección y adaptación en Scrum)
