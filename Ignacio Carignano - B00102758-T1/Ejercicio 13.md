## Resolución
1. ¿Qué es Extreme Programming (XP) y cuál es su objetivo principal dentro de las metodologías ágiles?

Extreme Programming (XP) es una metodología ágil de desarrollo de software enfocada en mejorar la calidad del software y la capacidad de adaptación a los cambios de requisitos.

Su objetivo principal es desarrollar software funcional mediante ciclos cortos, feedback constante, comunicación continua y prácticas técnicas que permitan responder rápidamente a las necesidades del cliente.

XP prioriza:

  - la colaboración
  - las pruebas
  - la simplicidad
  - la entrega continua de valor

    
2. ¿Cuáles son los cinco valores principales de XP? Explicá brevemente cada uno.

🔹 Comunicación

Busca mejorar el intercambio de información entre los integrantes del equipo y el cliente.

🔹 Simplicidad

Consiste en desarrollar únicamente lo necesario, evitando complejidad innecesaria.

🔹 Feedback

Permite detectar errores y mejorar continuamente mediante pruebas y retroalimentación constante.

🔹 Coraje

Implica tomar decisiones difíciles, aceptar cambios y refactorizar cuando sea necesario.

🔹 Respeto

Promueve el trabajo colaborativo y el reconocimiento entre todos los integrantes del equipo.


3. ¿Por qué XP considera que las pruebas son un elemento fundamental del desarrollo de software?

XP considera fundamentales las pruebas porque permiten detectar errores rápidamente y asegurar que el software funcione correctamente durante todo el desarrollo.

Las pruebas continuas ayudan a:

  - mantener la calidad
  - facilitar cambios
  - reducir errores en producción
  - generar confianza en el sistema

Además, XP propone realizar pruebas constantemente y automatizarlas siempre que sea posible.


4. ¿Qué es Test Driven Development (TDD) y cómo se relaciona con XP?

TDD (Test Driven Development) es una práctica donde primero se escriben las pruebas y después el código necesario para que esas pruebas funcionen.

El proceso generalmente sigue tres pasos:

  1- Crear una prueba.
  2- Escribir el código mínimo para aprobarla.
  3- Refactorizar el código.

TDD se relaciona con XP porque forma parte de sus prácticas principales y ayuda a mantener un software más confiable, simple y fácil de modificar.


5. ¿En qué consiste la práctica de Pair Programming? Mencioná dos ventajas y una posible dificultad.

Pair Programming consiste en que dos desarrolladores trabajen juntos en una misma computadora.

Uno escribe el código mientras el otro revisa, analiza y propone mejoras. Luego pueden intercambiar roles.

Ventajas:

  - Reduce errores y mejora la calidad del código.
  - Favorece el aprendizaje y la comunicación dentro del equipo.
  - 
Posible dificultad:

  - Puede resultar cansador o difícil de coordinar si las personas tienen distintas formas de trabajo.

    
6. ¿Qué son las historias de usuario en XP y por qué se prefieren frente a una especificación extensa de requisitos?

Las historias de usuario son descripciones simples y breves de funcionalidades vistas desde la perspectiva del usuario o cliente.

XP las prefiere porque:

  - son más fáciles de entender
  - permiten adaptarse rápidamente a cambios
  - evitan documentación excesiva

Además, favorecen la comunicación continua con el cliente y el desarrollo incremental.


7. ¿Qué significa Continuous Integration en XP y qué beneficios aporta al equipo de desarrollo?

Continuous Integration (Integración Continua) consiste en integrar frecuentemente el código desarrollado por todos los integrantes del equipo en una única versión compartida del sistema.

Sus beneficios son:

  - detectar errores rápidamente
  - evitar conflictos grandes de integración
  - mantener el sistema funcionando constantemente
  - mejorar la colaboración del equipo

    
8. ¿Cómo se aplica el concepto de Weekly Cycle en un proyecto desarrollado con XP?

El Weekly Cycle organiza el trabajo en ciclos semanales cortos.

Durante cada semana:

  - se planifican tareas
  - se desarrollan funcionalidades
  - se realizan pruebas
  - se entrega una versión funcional del software

Esto permite obtener feedback rápido y adaptarse continuamente a cambios o nuevas necesidades.


9. En XP se plantea que se fija el tiempo, el costo y la calidad, y se negocia el alcance. ¿Qué significa esta idea? Explicalo con un ejemplo.

Significa que:

  - el tiempo del proyecto
  - el presupuesto
  - la calidad esperada se mantienen estables, mientras que las funcionalidades pueden ajustarse según las prioridades
    
Ejemplo:

Si un equipo tiene 2 meses para desarrollar una aplicación y el cliente solicita demasiadas funcionalidades, en lugar de reducir calidad o extender tiempos, XP propone priorizar las funciones más importantes y dejar otras para futuras iteraciones.

De esta manera se garantiza entregar un producto funcional dentro del tiempo establecido.


10. Elegí tres prácticas de XP y explicá cómo podrían aplicarse en un proyecto real de desarrollo de software.
    
🔹 Pair Programming

Dos desarrolladores trabajan juntos implementando funcionalidades críticas para reducir errores y compartir conocimiento.

🔹 Continuous Integration

Cada cambio realizado en el código se integra automáticamente al proyecto para detectar problemas rápidamente.

🔹 TDD

Antes de programar una funcionalidad se crean pruebas automáticas para asegurar que el sistema funcione correctamente desde el inicio.

## Conclusión

XP propone un enfoque ágil centrado en la comunicación, el feedback continuo, la simplicidad y la calidad del software. Sus prácticas permiten desarrollar sistemas más flexibles, mantenibles y adaptados a las necesidades reales del cliente.
