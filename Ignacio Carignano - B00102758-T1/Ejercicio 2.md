# Ejercicio Nro: 2

## Enunciado
¿Qué es para vosotros el desarrollo de software?

1. ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
2. ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
3. ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
4. ¿Cuándo se ha desarrollado un buen software?


## Resolución

1) Un ejemplo de desarrollo de software puede ser la creación de una aplicación o sistema que atraviesa todas las etapas del ciclo de vida, desde la idea inicial hasta su distribución final.

En los materiales se menciona, por ejemplo, el desarrollo de un videojuego, donde se pasa por etapas como:

- Concepto y prototipo
- Producción (desarrollo de funcionalidades)
- Pruebas (Alpha y Beta)
- Versión final y distribución

Esto refleja cómo un software se construye de forma progresiva, validando cada etapa antes de avanzar.


2) El desarrollador tiene un rol clave dentro del proyecto, ya que es quien se encarga de implementar y construir el software en la etapa de desarrollo.

Durante el ciclo de vida, su trabajo se centra principalmente en:

- Programar las funcionalidades
- Participar en la creación de prototipos
- Ajustarse a los requerimientos del proyecto
- Colaborar en las pruebas y mejoras

Además, el desarrollador forma parte del equipo que se adapta a las necesidades del proyecto a medida que este evoluciona


3) Sí, dentro de un proyecto de software existen distintos roles necesarios para su correcto funcionamiento.

Según el material, una metodología permite definir claramente:

- Equipos de trabajo
- Roles y responsabilidades
- Planificación y objetivos

Esto implica que, además de desarrolladores, participan otras personas como:

- Líderes o responsables del proyecto
- Personas que analizan requisitos
- Equipos de prueba
- Stakeholders o interesados

Cada uno cumple una función específica dentro del proceso.


4) Se puede considerar que un software es bueno cuando cumple con los objetivos planteados desde el inicio del proyecto y logra equilibrar correctamente los factores principales del desarrollo:

- Calidad
- Tiempo
- Presupuesto

Además, un buen software:

- Funciona correctamente
- Cumple con los requisitos definidos
- Ha sido probado adecuadamente
- Puede ser utilizado por los usuarios sin problemas
